# Learning Knowledge Base v0.1.0 候选发布说明

## 定位

v0.1.0 是 Learning Knowledge Base 的首个可本地使用与演示的候选版本，适合个人学习记录、课程组织和可追溯知识整理。它不是 1.0 稳定承诺，也不提供云同步、多人协作或服务端账户体系。

## 核心能力

- 本地 Markdown 笔记、标签、目录、专题/项目与课程章节；
- 标题和正文搜索、Wiki 链接、前向/反向链接及随机回顾；
- 年度笔记创建足迹，支持年份切换、键盘访问和按日期打开笔记；
- 笔记图谱，以及只读、approved-only 的实体图谱和实体详情导航；
- AI 整理预览与知识实体/关系候选审核：用户确认后才会写入笔记或知识模型；
- AI 历史、来源和知识审计浏览；
- Backup v5 JSON 导出、验证、合并恢复与本地目录 latest/每日快照备份；
- 隔离演示数据、产品截图和生产构建验收流程。

## 本次发布聚焦

本候选版本完成了编辑工作区、年度创建足迹、轻量 NoteProjection、实体图谱加载与工具栏可读性、性能基线、Microsoft Edge 手工验收，以及 README 与演示截图的整理。它聚焦于稳定本地体验，而不是新增未经真实使用验证的业务模块。

## 从源码运行

```bash
npm install
npm run dev
npm run build
npm run preview
```

Windows 的完整本地模式还可使用仓库内的启动脚本或 `npm run local`。更多说明见 [README](../../README.md#快速开始)。

## 数据与隐私

- 数据默认存放在当前浏览器 Profile、当前站点 origin 对应的 IndexedDB 中；清除浏览器站点数据可能导致本地数据丢失，请先导出 Backup；
- 建议定期导出 JSON Backup，并按需要启用本地目录备份；
- AI 仅在用户主动触发时工作；API Key 只由本机 Node 服务读取，浏览器不保存或发送 Key；
- 未配置 AI 时，笔记、搜索、媒体、图谱和备份等非 AI 功能仍可使用。

- 26 项 Node 契约测试；
- 3 项发布版本契约测试；
- 33 项生产 Playwright E2E；

- 发布前依赖审计已完成，生产依赖与完整依赖树均未发现 npm audit 漏洞。

验收边界和证据见 [产品体验升级生产验收报告](../superpowers/audits/2026-07-18-product-experience-upgrade-acceptance.md)。

## 已知限制

- 数据主要位于单个浏览器 Profile，尚无云同步或多人协作；
- 正文搜索和部分孤立链接检查仍可能执行按需 O(N) 扫描；
- 300 实体图谱属于较重页面，不保证所有设备表现相同；
- 当前没有数小时/数天级别的 Heap 泄漏证明；
- html2pdf 保留既有的大 chunk 构建警告；
- 验收覆盖 Chromium 与一套 Edge 环境，不代表所有浏览器、设备或企业合规场景。

## 升级与恢复

这是首个候选发布，不承诺可从所有更早的开发快照无缝迁移。升级或切换浏览器 Profile 前，请先导出并验证 Backup；不要通过手工修改 IndexedDB 或备份 JSON 规避恢复校验。此次候选发布不改变 Dexie 数据库版本或 Backup 数据格式。
