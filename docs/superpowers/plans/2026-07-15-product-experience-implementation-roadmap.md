# 产品体验升级实施路线

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development or superpowers:executing-plans to implement tasks one at a time. Each task requires an independent review gate.

**Goal:** 在保持 Markdown、本地数据和现有知识能力兼容的前提下，提升编辑、笔记创建足迹、页面布局和长期性能体验。

**Architecture:** 保留 CodeMirror 和现有 Dexie 数据模型，先修正正确性与渲染边界，再逐步建立编辑器辅助面板、年度笔记足迹和性能治理。真实学习活动与计时数据另立模型设计阶段。

**Tech Stack:** React、TypeScript、CodeMirror 6、Zustand、Dexie、React Flow、Vitest、Playwright、Vite。

## Global Constraints

- Markdown 继续作为唯一正文数据源。
- 不替换 CodeMirror。
- 第一至第四阶段不升级 Dexie schema。
- 第一至第四阶段不升级 Backup。
- 不继续扩张 AI 能力。
- 性能优化必须先有代码证据或测量基线。
- 每个任务只完成一个可独立验收目标。
- UI 改造必须保护既有本地数据、自动保存和 Markdown 导入导出兼容性。

---

## 1. 输入设计与审计结论

### 已确认的编辑器事实

- 当前是 CodeMirror 6 Markdown 源码编辑器，Markdown 是唯一正文事实来源；800ms 合并防抖保存，AI 应用与编辑器 pending/in-flight 保存已由 `editorSaveCoordinator` 协调。
- 不替换 CodeMirror。每次正文变化会生成完整字符串；250KiB 正文的短输入 Chromium 中位数约 37.1ms，属于已测得的高频风险方向。
- 编辑模式的 AI 整理、AI 知识分析、知识结构和 AI 历史目前直列在正文之前；尚无右侧抽屉或停靠面板。
- Wiki/双链派生查询存在高频全笔记扫描风险；必须先收敛查询边界，再做编辑器视觉迁移。

### 已确认的热力图事实

- 当前数据只表示活动表 `Note.createdAt` 的创建足迹，按 UTC 日期聚合，窗口为最近 20 或 26 周。
- “学习热力图”超出了当前统计能力；星期布局、自然周、统计窗口和日期语义是 P1 正确性问题。
- 140/182 个固定 DOM 方格不是性能瓶颈；完整学习时长、练习、复盘和综合强度需要未来独立模型。

### 已确认的性能事实

- html2pdf、CodeMirror、实体图谱都已延迟加载，图谱不是首屏直接加载；不应为了猜测继续细碎拆分基础模块。
- `allNotes` 长期保存完整活动笔记正文。100 篇规模良好，500 篇可接受，2000 篇真实长正文存在结构性内存风险。
- 300 approved 实体图谱首次可见约 1.25 秒、可见时约 7022 DOM 节点；十轮页面循环未发现持续 DOM/listener 增长，且没有 heap 级泄漏结论。
- 暂不实施全局 React.memo、Canvas 热力图、Worker/WASM 或替换 CodeMirror。

## 2. 总体实施原则

1. 数据正确性优先于视觉美化，尤其是本地日期、自然周和统计文案。
2. 高频编辑体验优先于低频图谱、导出和设置页优化。
3. 只处理已有代码或测量证据支持的问题；理论风险先以诊断任务验证。
4. 先建立组件、查询和状态边界，再迁移复杂视觉内容。
5. 不一次性重写编辑器，不创建第二份正文，也不把 Markdown 转换为块 JSON。
6. 没有真实学习活动数据时，只展示诚实的“笔记创建足迹”。
7. 第一至第四阶段不改 schema 或 Backup；长期学习记录必须独立完成模型与兼容性设计。
8. 每个任务一个主要提交、一个独立验收结果；不得混合正确性、视觉和持久化迁移。

## 3. 阶段路线

| 阶段 | 目的 | 进入条件 | 退出条件 |
|---|---|---|---|
| 一：正确性与高频性能 | 修正热力图日期口径、收敛编辑输入与链接查询 | 当前基线文档已提交 | 既有数据语义正确，长正文与链接回归受保护 |
| 二：编辑器布局与视觉 | 正文优先、保存可见、右侧辅助面板 | 阶段一编辑边界稳定 | 原有 AI/知识入口均保留且可访问 |
| 三：专业笔记创建足迹 | 在 `Note.createdAt` 范围内完成年度体验 | 热力图日期契约已固定 | 年度、键盘、tooltip、日期导航和 E2E 完成 |
| 四：性能与长期稳定 | 治理正文常驻、图谱上限、基线与备份大资产 | 有可重复测量/诊断输入 | 关键趋势可复测，优化不破坏本地数据 |
| 五：学习活动基础设施 | 课程、会话、复盘、掌握度及真正学习热力图 | 独立模型设计批准 | 不属于近期实施范围 |

## 9. 性能预算

预算用于同环境趋势比较，不是 CI 硬门槛；Chromium 基线不能替代真实8GB Edge验收。

| 场景 | 当前参考 | 近期目标 |
|---|---:|---|
| 编辑器5/50/250KiB短输入 | 13.0/26.0/37.1ms | 不高于当前稳定范围，且不增加保存竞态 |
| 编辑器自动保存 | 约820ms，含800ms debounce | 保持约700–1000ms合并行为，页面切换必须最终flush |
| 热力图100/500/2000笔记 | 579.8/608.8/583.5ms首次可见 | 年度365/366格不引入明显随格数线性劣化 |
| 热力图年份切换/日期详情 | 未建立 | 在D1后记录可比较基线，不用sleep断言 |
| 图谱50/300实体 | 782.5/1252.0ms首次可见 | 300实体不明显恶化，离开后DOM/listener应回落至稳定页面范围 |

## 10. 测试策略

- 日期：纯函数覆盖本地时区、闰年、自然周、年边界和窗口一致性。
- 编辑：5/50/250KiB确定性输入、保存最终flush、AI应用竞态、链接查询取消和切换笔记。
- 面板：打开/关闭、键盘可访问、辅助功能入口、收起后的状态保持。
- 足迹：年度聚合、年份切换、tooltip、键盘、日期导航、空/加载/错误状态。
- 生产 E2E：真实 IndexedDB 初始化后写入合法数据；`onupgradeneeded` 必须失败，禁止测试自行建表。
- 生命周期：反复进出编辑器、热力图与图谱，检查 DOM/listener 趋势；不能把不可控 sleep 作为核心断言。
- 人工：真实8GB Windows Edge 验证中文 IME、大段粘贴、图片/AI历史备份、300实体交互和长时间切换。

## 11. 风险与回退

| 阶段 | 最大风险 | 回退与兼容 | 是否需要 feature flag / E2E |
|---|---|---|---|
| 一 | 输入/链接优化破坏自动保存或结果时序；日期修复改变用户认知 | 保留 Markdown、Note字段和保存协调器；错误可回退单个实现提交 | 不需要 flag；编辑路径需生产E2E |
| 二 | 面板迁移让已有 AI/知识入口不可达 | 容器与内容分两次提交；可删除面板 UI 回到既有直列入口 | 不需要 flag；面板可访问性测试 |
| 三 | 新视图继续使用误导统计或日期偏差 | 只消费 `createdAt`；可删除年度 UI，不影响笔记数据 | 需要生产E2E |
| 四 | Store治理影响搜索、首页、热力图或切换；图谱优化破坏导航 | 先D1基线，再最小改动；保持读取/业务契约与节点上限 | 必须关键E2E和真实Edge手工验证 |

CodeMirror 不得无计划替换；热力图口径修正后不得继续使用“学习时长/强度”文案；`allNotes` 治理不得退化搜索、首页、热力图或笔记切换；辅助面板不得隐藏无替代入口。
