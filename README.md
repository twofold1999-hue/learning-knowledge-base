# 外接知识库 / Personal Learning OS

本地优先的个人学习与知识管理工具。把自由笔记、学习单元、视频进度、知识关联、回顾和导出放在同一个工作台中，适合技术课程、考公、阅读、书法、绘画、心理学等长期学习主题。

> 默认不上传笔记；不会读取、保存或上传 B 站 Cookie；不会代理或转码视频流。

## 功能总览

| 模块 | 已实现能力 |
| --- | --- |
| 自由记录 | Markdown 编辑、代码块、图片、标签、目录、专题项目、双向链接与关联概念。 |
| 学习单元 | 用课程、书籍、题库或训练计划的章节管理顺序、完成状态和续播点。 |
| 统一学习媒体 | 每个章节可继承计划默认来源，也可独立使用 B 站链接、HTTPS 视频直链、`/media/` 媒体库或临时本地文件。 |
| 视频注释 | 自动保存续播进度；可把“19:00–21:00｜软件安装过程”插入正文并保留链接。 |
| B 站学习 | 网页内官方外链播放器；浏览器扩展可读取真实 B 站播放时间，并支持高清悬浮学习。 |
| 回顾与组织 | 学习热力图、随机回顾、知识图谱、搜索、目录/专题/课程页与拖拽章节排序。 |
| 数据保护 | IndexedDB 本地存储、本地目录备份、JSON 备份恢复、回收站、Markdown/PDF/Word 导出。 |

## 快速开始

1. 安装 [Node.js 24 LTS](https://nodejs.org/)（项目通过 `.nvmrc` 固定主版本）。
2. 双击 `启动知识库.cmd`。
3. 浏览器会自动打开 `http://127.0.0.1:4173`。
4. 结束时双击 `停止知识库.cmd`。

不需要手动打开终端。更新源码或拉取新版后再次双击启动脚本，它会重建网页。若仍显示旧界面，按 `Ctrl + Shift + R` 强制刷新；不要清除站点数据，以免误删本地笔记。

## 本地 AI（DeepSeek）

AI 请求通过同机 `127.0.0.1` Node 服务代理，浏览器不会读取、保存或发送 API Key。首次使用时，在项目根目录手动创建 `.env.local`：

```dotenv
DEEPSEEK_API_KEY=你的新Key
DEEPSEEK_MODEL=deepseek-v4-flash
DEEPSEEK_BASE_URL=https://api.deepseek.com
```

保存后重新双击 `启动知识库.cmd`。旧版 `VITE_DEEPSEEK_*` 配置不再生效；请手动迁移变量名，并在确认代理可用后轮换曾经暴露给浏览器构建的旧 Key。不要分享 `.env.local`，也不要把 Key 发到聊天、备份或 Git。
## 学习工作流

### 记录方式

- **自由笔记**：知识点、灵感、错题、练习心得、阅读摘录等。
- **学习单元**：一门课的一章、一本书的一节、一个题型或一次训练；可以关联媒体与学习进度。

学习计划可填写默认媒体地址；具体章节可覆盖默认值。因此同一计划可共用一套课程视频，也可让每章绑定不同素材。

### 统一学习媒体区

打开学习单元后点击“学习媒体”，可使用：

- **B 站链接**：使用 B 站官方外链播放器，知识库页面不会被画质/登录提示带走。
- **HTTPS 视频直链**：按原始地址播放，应用不二次压缩或转码。
- **本地媒体库**：把 `mp4`、`webm` 等文件放到 `media/`，填写 `/media/文件名.mp4`。
- **临时本地文件**：直接选择电脑中的视频；只在当前页面有效，刷新后需要重新选择。

本地/直链视频每约 8 秒、暂停和播放结束时都会保存续播点，还可点击“悬浮播放”进入浏览器画中画。

### 视频片段笔记

对本地/直链视频：在知识点开始处点击“设为起点”，到结束处输入旁注并点击“插入片段注解”。正文会得到：

```md
- 🎬 [19:00–21:00](视频链接)｜这里讲解 xx 软件的安装过程
```

片段末尾会同时作为下次续播位置保存。

## B 站高清悬浮学习（可选）

网页 iframe 不能可靠继承 B 站主站登录态和最高画质权限。本项目不通过抓 Cookie、视频代理或绕过平台限制来解决；需要账号画质时，直接用真实 B 站页面播放，扩展只把它与当前笔记连接起来。

### 安装扩展

1. 启动知识库。
2. 在 Chrome 或 Edge 打开 `chrome://extensions` / `edge://extensions`。
3. 开启“开发者模式”，点击“加载已解压的扩展程序”。
4. 选择项目中的 `browser-extension/` 文件夹。
5. 将“知识库 B 站学习助手”固定到工具栏。

扩展当前版本为 `0.2.0`。项目更新后，在扩展管理页点击一次“重新加载”。

### 使用流程

1. 在章节的媒体区点击“高清悬浮学习”。
2. 扩展打开真实 B 站页面；在该页登录并选择可用清晰度。
3. 点击右下角的“进入画中画并返回笔记”。
4. 视频进入系统悬浮窗，扩展自动切回发起的知识库章节。
5. 在扩展右侧栏保存进度、插入单点或片段旁注。

浏览器要求画中画必须由一次真实点击触发，所以第 3 步不能静默自动化。画中画窗口通常只保留播放/暂停等基础控件；画质、弹幕、倍速等仍在原始 B 站页调整。扩展只访问 B 站页面和本地知识库地址，不读取或保存 Cookie。

## 数据、备份与导出

笔记、目录、课程、项目、图片索引和回收站保存在浏览器 IndexedDB 数据库 `LearningKnowledgeBase` 中。它与当前浏览器配置文件及 `127.0.0.1:4173` 地址绑定，不是项目文件夹中的明文笔记。

- 更新和重启应用不会删除笔记；不同浏览器、不同电脑默认互不共享。
- 不要在浏览器设置中清除这个站点的数据。
- 建议在“设置”中配置本地目录备份，并定期导出 JSON 备份。
- 删除笔记会先进入回收站；JSON 备份用于完整迁移或恢复。
- 可导出 Markdown、PDF 和 Word。推荐同时保留 Markdown/JSON 作为长期归档原件。

## 分享给其他人

将以下内容压缩发送：

- `dist/`
- `scripts/local-server.mjs`
- `启动知识库.cmd`、`停止知识库.cmd`
- 可选：`media/`（需要附带本地课程视频时）

接收者安装 Node.js LTS 后双击启动即可。每个人的数据保存在自己的浏览器中，不会与你同步，也不会泄露给你；这不是云同步方案。

## 开发与测试

```bash
npm install
npm run dev          # Vite 开发服务
npm run typecheck    # TypeScript 检查
npm run test         # Vitest 单元测试
npm run build        # 生产构建
npm run check        # 类型检查 + 单元测试 + 构建
npm run check:all    # 类型检查 + 单元测试 + 构建 + 生产 E2E
npm run test:e2e     # Playwright Chromium 冒烟测试
```

项目已安装 `@playwright/test` 和 Chromium 测试运行时。新电脑首次运行端到端测试时执行：

```bash
npx playwright install chromium
```

Playwright E2E 会先构建 `dist/`，再以独立的 `127.0.0.1:4174` Node 静态服务器运行；它不会使用 Vite 开发服务、不会触碰正式 `4173` 实例或 `.runtime/local-server.json`，也不需要或调用任何 DeepSeek API。测试覆盖首页、SPA 直达路由、安全响应头、健康检查和 IndexedDB 笔记持久化。

GitHub Actions 会在推送到 `main`、面向 `main` 的 Pull Request 以及手动触发时，执行类型检查、单元/Node 服务测试、生产构建和 Chromium E2E。

## 项目结构与安全边界

```text
src/components/       编辑器、媒体面板、热力图、导航等组件
src/pages/            首页、编辑、课程、专题、图谱、热力图、设置
src/services/         IndexedDB、备份、导出、媒体、搜索、链接、校验
src/stores/           Zustand 状态管理
browser-extension/    B 站时间点与高清悬浮学习扩展
scripts/              本地服务、AI 同源代理（含视频 Range 请求）
media/                可选本地视频媒体库
tests/e2e/            Playwright 浏览器冒烟测试
src-tauri/            可选桌面端壳工程
```

- Markdown 渲染前会净化不可信 HTML。
- 持久媒体地址仅接受 HTTPS、`/media/` 或本机地址，不接受 `javascript:`、`data:` 与任意远程 HTTP。
- 本地媒体服务器限制路径在 `dist/` 与 `media/` 内，并支持 Range 请求，便于拖动和续播。
- 页面配置 CSP、`nosniff`、引用来源和权限策略响应头。
- B 站账号、会员、地区、版权与可选清晰度由 B 站官方页面决定。
